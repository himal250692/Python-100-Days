### Python Application Areas and Career Paths

Python is an elegant, explicit, and simple programming language.

- Low learning curve; accessible to non-CS learners.
- Open-source ecosystem with rich libraries and frameworks.
- Interpreted language with excellent cross-platform portability.
- Dynamically typed; supports object-oriented and functional programming.
- High code readability with strong style conventions.

Python is widely used in:

- Backend development - Python / Java / Go / PHP
- DevOps - Python / Shell / Ruby
- Data acquisition - Python / C++ / Java
- Quantitative trading - Python / C++ / R
- Data science - Python / R / Julia / Matlab
- Machine learning - Python / R / C++ / Julia
- Automated testing - Python / Shell

As a Python developer, you can choose from many career directions:

- Python backend engineer (servers, cloud platforms, data APIs)
- Python operations engineer (automation, SRE, DevOps)
- Python data analyst (analysis, BI, digital operations)
- Python data scientist (ML, deep learning, algorithms)
- Python web crawler engineer (not recommended as a long-term track)
- Python test engineer (automated testing, test development)

> **Note**: Data science is currently one of the hottest tracks. Both internet companies and traditional industries have accumulated massive data. Organizations need data scientists to discover business value and support data-driven decision making.

A few suggestions for beginners:

- **Make English your working language.**
- **Practice makes perfect.**
- **All experience comes from the mistakes you've made.**
- **Don't be a freeloader.**
- **Embrace AI and improve efficiency.**

#### Day91 - [Team Project Development: Problems and Solutions](./Day91-100/91.团队项目开发的问题和解决方案.md)

1. Software process models
   - Classical process model (Waterfall)
     - Feasibility analysis (whether to do it), output: feasibility report.
     - Requirements analysis (what to build), output: requirements specification and UI prototype.
     - High-level and detailed design, output: ER diagrams, physical models, class diagrams, sequence diagrams, etc.
     - Coding/testing.
     - Release/maintenance.

     The biggest weakness of Waterfall is poor adaptability to changing requirements.

   - Agile development (Scrum) - Product Owner, Scrum Master, Developers - Sprint
     - Product backlog (user stories, prototypes)
     - Planning meetings (estimation and budgeting)
     - Daily development (standups, pomodoro, pair programming, test-first, refactoring)
     - Bug fixing (description, reproduction steps, tester, assignee)
     - Release
     - Review meeting (showcase with users)
     - Retrospective

     > Agile Manifesto:
     > - **Individuals and interactions** over processes and tools
     > - **Working software** over comprehensive documentation
     > - **Customer collaboration** over contract negotiation
     > - **Responding to change** over following a plan

     > Roles: Product owner (decides what to build), team lead/Scrum master (removes blockers and protects the team), development team (developers and testers).

##### Topic Selection and Business Understanding

1. Scope definition
   - CMS (user side): news aggregators, Q&A/sharing communities, movie/book review sites, etc.
   - MIS (user + admin): KMS, KPI systems, HRS, CRM, supply chain, warehouse management, etc.
   - App backend (admin + APIs): second-hand trading, magazines, niche e-commerce, news, travel, social, reading apps, etc.
   - Other types: choose domains aligned with your background and controllable business complexity.

2. Requirements understanding, module decomposition, and task assignment
   - Understand requirements with brainstorming and competitor analysis.
   - Draw a mind map (XMind): each module is a branch, each feature is a leaf node (verb-based).
   - Assign tasks based on priority, importance, and workload.
